## 🚀 实际使用场景

### 场景1: Android 用户首次启动

```
1. 下载并安装 APK
   ↓
2. 打开应用
   ↓
3. 自动加载 6 个源 (10-30秒)
   提示: "加载源 1/6", "加载源 2/6" ...
   ↓
4. 显示 800-1000+ 个频道
   ↓
5. 选择频道播放
   - 如果黑屏/卡顿 → 1.5-3秒自动切换
   - WiFi: 3秒内切换
   - 移动网络: 1.5秒快速切换
   ↓
6. 完美全屏观看 (Home 条不挤占)
```

### 场景2: iOS 用户遇到不可用频道

```
1. 打开应用，选择频道
   ↓
2. [央视1] 线路1 → 黑屏
   ↓ 1.5秒检测
3. 提示: "线路失败，切换下一线路 (2/3)"
   自动切换到线路2
   ↓
4. 线路2 → 还是黑屏
   ↓ 1.5秒检测
5. 提示: "线路失败，切换下一线路 (3/3)"
   自动切换到线路3
   ↓
6. 线路3 → 还是黑屏
   ↓ 1.5秒检测
7. 提示: "当前频道所有线路不可用，切换下一频道"
   ↓ 0.8秒延迟
8. 自动切换到 [央视2]
   ↓
9. [央视2] 线路1 → 成功播放 ✅
```

**总耗时**: ~5秒自动找到可用频道

## 📞 用户支持

### 问题反馈
- GitHub Issues: https://github.com/wfygefjgd/live-player/issues
- 讨论区: https://github.com/wfygefjgd/live-player/discussions

### 常见问题

**Q1: Android 为什么首次加载需要 10-30 秒？**
A: 因为需要从 6 个不同的 GitHub 源加载频道列表，并进行合并和去重。加载完成后会缓存，下次启动会很快。

**Q2: iOS 为什么还是偶尔会卡一下？**
A: 如果源质量不好，可能需要多次切换才能找到可用线路。v1.4.7 已经将检测时间缩短到最快 1.5 秒，会自动快速切换。

**Q3: 为什么有些频道一直切换？**
A: 说明这个频道的所有线路都不可用。系统会自动切换到下一个频道。

**Q4: iOS 版本在哪里下载？**
A: iOS 需要等待 GitHub Actions 自动构建完成。构建完成后会出现在：
https://github.com/wfygefjgd/live-player/releases/tag/v1.4.7-ios
